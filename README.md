# Schedule Master

**"一个做不好任务记录的知识管理系统不是一个好的调度工具"**

"A system that fails at capturing tasks will never succeed at scheduling them."

Scheduling is merely the organized capture of the unfinished.

Schedule Master 坚信：只有当一个系统能让人“无感地记录个人琐事”时，它才具备做“任务排期”的资格。任务排期本质上就是对**尚未完成的人物与琐事**进行的及时记录与位置重排。我们将录入的极简感与调度的全局感统一在了一张“单元格网格”中。

### 🚀 核心特色

#### 1. 琐事记录的“无感化” (Frictionless Entry)

* **单元格即记事本：** 告别“新建任务”弹窗。在任何格子里直接打字，回车即存。这种像 Excel 般顺滑的体验，是为了让用户在产生念头的瞬间就完成记录。
* **从琐事到任务的自然演进：** 你随手记下的一行文字，在网格中通过简单的拖拽拉长，就从一条“琐事”进化成了一个有时间跨度、有权重的“任务”。
* **记录的成就感：** 实时划掉已完成的琐事，伴随清脆的交互反馈，让“记事”这件事本身变得解压且上瘾。

#### 2. “二维矩阵”驱动的调度视野 (Grid-Based Scheduling)

* **任务的物理占位：** 每一个任务在网格中都有其明确的“物理坐标”（人、时间）。这种布局能迫使管理者直观意识到：**时间是有限的资源**。
* **留白的力量：** 通过单元格的空白，你可以一眼识破谁的日程还有空余，而不是在繁琐的任务列表里计算工时。
* **冲突的显性化：** 当多个任务在同一时间挤在同一个格子里，系统会自动伸展轨道平铺显示。这种“任务打架”的视觉提示，会倒逼你立即进行重录或排期优化。

#### 3. 极简的工业美学与交互

* **侧边栏（Sidebar）视图：** 专注于“个人录入”。去掉复杂的时间轴，它就是一个常驻屏幕边缘的极简备忘录。
* **全屏（Full-grid）视图：** 专注于“团队规划”。展开后，所有成员记录的琐事汇聚成一张清晰的时间作战大地图。
* **局域网同步：** 基于 P2P 的实时互联，让团队成员之间的任务流动像在同一张纸上涂改一样简单。

### 💡 核心交互逻辑：从“随手记”到“大排期”

在 **Schedule Master** 中，记录一个任务不再是填表，而是一个自然流转的过程。以下是用户在 3 秒内完成操作的典型场景：

#### 第一阶段：无感记录 (Capturing - 1秒)

用户在侧边栏模式（Sidebar）下，鼠标单击当前行，直接输入文字：

* **动作：** 单击 -> 输入 "完成技术方案初稿" -> 回车。
* **结果：** 此时，它仅仅是一个占据了当前时间格（1小时）的**个人琐事块**。它不干扰他人，只是你自己的待办。

#### 第二阶段：定义跨度 (Defining - 1秒)

用户意识到这个任务需要更长时间：

* **动作：** 鼠标悬停在任务块右边缘，向右拖拽拉长 3 个格子。
* **结果：** 琐事块在视觉上变成了**任务条**，横跨了 4 小时。由于是在二维网格中操作，你可以清晰看到这 4 小时内你是否还记录了其他琐事（如“取快递”、“周会”）。

#### 第三阶段：智能调度 (Scheduling - 1秒)

用户决定将此任务正式排入团队计划：

* **动作：** 展开全屏视图，将该任务块从自己的行拖拽到“项目 A”的公用行。
* **结果：** * **自动对齐：** 任务块自动吸附到网格，确保时间分秒不差。
* **冲突处理：** 如果目标位置已有其他任务，系统自动垂直平铺显示，并发出轻微的视觉发光预警。
* **实时同步：** 局域网内的其他成员瞬间在他们的全景图上看到了这个“新方块”。

### 📈 MVP (最小可行性产品) 开发路线图

为了实现上述体验，建议按以下优先级进行开发：

1. **Phase 1 (基础网格):** 实现基于 `(row, col)` 索引的 PyQt6 绘制逻辑，确保任务块“格对格”吸附。
2. **Phase 2 (即时录入):** 实现单击单元格直接弹出透明输入框，按下 Enter 转换成色块。
3. **Phase 3 (平铺算法):** 编写“多轨避让”逻辑，解决当任务时间重叠时的视觉展示问题。
4. **Phase 4 (视图切换):** 增加侧边栏模式与全屏模式的平滑过渡动画。
5. **Phase 5 (同步引擎):** 接入 Zeroconf 局域网广播，实现多机位的数据即时交换。
