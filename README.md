# GridFlow: 游戏化单元格任务调度系统

### 🌟 项目愿景 (The Vision)

**GridFlow** 并非传统的任务列表，而是一个基于“二维时间矩阵”的资源调度器。它的核心理念是：**像记琐事一样简单录入，像排日程表一样直观调度。**

我们打破了“列表式”工具的碎片化，通过 Excel 般的全局视野，让管理者一眼看清“谁在忙、谁在等、任务在哪里打架”。

---

### 🎨 核心设计哲学

#### 1. Excel 式的编辑体验 (The Excel-Like Feel)

* **网格即容器：** 抛弃繁琐的弹出表单。点击网格，直接打字。
* **强制吸附与对齐：** 任务不再是漂浮的进度条，而是严格对齐到“人-时间”单元格的填充块。
* **所见即所得：** 添加任务时，你能直观意识到谁的日程还有留白，而非在被遮挡的侧边栏中猜测。

#### 2. 《缺氧》风格的工业美学 (ONI-Inspired UI)

* **视觉语言：** 采用深色工业风格，配合像素风或等宽字体，利用发光边框表示选中状态。
* **游戏化反馈：** 任务完成时，覆盖类似游戏里的斜纹遮罩；录入和完成动作伴随清脆的机械声效（ASMR）。
* **动态理顺：** 即使在同一格发生任务重叠，系统也会像游戏逻辑一样自动“分轨”平铺，绝不遮挡信息。

#### 3. 极简录入与无感同步 (Frictionless Workflow)

* **碎片化收集：** 鼓励记录琐事。个人琐事是任务排期的基础，只有最全的记录，才有最准的调度。
* **局域网互联：** 采用 P2P 自动发现技术，无需配置服务器，团队成员即可在同一局域网内实时看到排期变动。

### 🛠️ 关键功能模块

| 模块 | 说明 |
| --- | --- |
| **双模视图 (Dual-View)** | **侧边栏模式**用于个人无感记录琐事；**全屏模式**用于团队资源全局调度。 |
| **自动理顺算法** | 独创的“单人多轨”布局，当任务时间冲突时自动垂直伸展，直观暴露“过载”风险。 |
| **语义化录入** | 点击格子直接输入，系统自动解析时间、分类及优先级，转化为可视化的色块。 |
| **同步中心** | 基于局域网广播的实时同步，确保 A 记录的任务，B 的屏幕上立刻跳出对应的方格。 |

---

### 🚀 为什么选择 GridFlow？

传统的管理软件（如 Jira/Trello）往往让用户感到在“填表格”，而 GridFlow 让用户感到在“管理系统”。

* **对于个人：** 它是一个解压、好玩的琐事本。
* **对于团队：** 它是一张永不混乱、实时更新的作战大地图。

---

### 🛠 技术栈概览

* **语言:** Python 3.10+
* **UI 框架:** PyQt6 (自定义绘图实现高性能网格渲染)
* **局域网通讯:** Zeroconf (自动发现) + Fast API / WebSockets
* **数据存储:** SQLite (本地优先，实时同步)
